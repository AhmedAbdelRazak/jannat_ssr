Purpose
This Next.js app is the server-rendered public Jannat Booking website. It was built to preserve the customer-facing structure and spirit of the legacy jannatbooking_frontend SPA while improving SEO, performance, crawlability, trust signals, RTL support, analytics, hotel discovery, chat/support, payments, and responsive UI.

Current Runtime
- Framework: Next.js App Router.
- React: React 19.
- Styling: app/globals.css.
- Local dev command: npm run dev.
- Local dev port: 3104.
- Production build command: npm run build.
- Production start command: npm run start.
- Package file: package.json.
- Main config: next.config.mjs.
- Request proxy/middleware: proxy.js.

Important Environment Variables
- NEXT_PUBLIC_SITE_URL
  - Public canonical site URL. Defaults are in lib/constants.js.
- NEXT_PUBLIC_API_URL / API_URL / SERVER_API_URL
  - Backend API base. lib/api.js defaults to https://xhotelpro.com/api.
- NEXT_PUBLIC_DEV_API_URL
  - Optional local backend API fallback in development. Default: http://localhost:8080/api.
- API_FETCH_TIMEOUT_MS
  - Server fetch timeout in milliseconds. Default: 3500.
- NEXT_PUBLIC_SOCKET_URL
  - Socket.IO base URL for support chat. Defaults from API base.
- NEXT_PUBLIC_GOOGLE_ANALYTICS_ID / NEXT_PUBLIC_GA_ID
  - Google Analytics measurement id.
- NEXT_PUBLIC_FACEBOOK_PIXEL_ID
  - Facebook Pixel id. Default exists in lib/constants.js.
- NEXT_PUBLIC_PAYPAL_CLIENT_ID_LIVE
  - Live PayPal client id for checkout.
- NEXT_PUBLIC_PAYPAL_CLIENT_ID_SANDBOX
  - Sandbox PayPal client id for checkout.
- NEXT_PUBLIC_PAYPAL_ENV
  - Checkout environment switch.
- NEXT_PUBLIC_GOOGLE_CLIENT_ID
  - Google login client id.

Support Chat
- components/SupportWidget.js
  - Public B2C support widget.
  - Uses /api/support-cases/new to create a case.
  - Uses /api/support-cases/client/:id to read/update public client-side case messages.
  - Uses Socket.IO for receiveMessage, typing, stopTyping, and closeCase events.
  - Polls the case while open as a fallback.
  - Query params are synced when the chat opens or when field changes are committed.
  - Important query params:
    - chat=open
    - chatName
    - chatContact
    - chatHotelId
    - chatHotelName
    - chatInquiry
    - chatDetails
    - chatLanguage
  - Best-practice behavior:
    - Do not update URL while the user is actively typing.
    - Commit URL changes when the user opens chat, changes fields, selects options, or starts chat.
    - Avoid loops by using isolated helpers in lib/chatQueryParams.js.
  - Related helpers:
    - lib/support.js
    - lib/chatQueryParams.js

PageSpeed And Agentic Update - 2026-06-19
- /llms.txt is served as Markdown and uses real Markdown links for the main public pages, sitemap, robots policy, offers, and representative hotel pages.
- Header language flags use tiny SVG assets instead of oversized PNG files.
- Flag assets receive long immutable cache headers.
- Next image device sizes include 384px and 512px candidates so mobile hotel cards can avoid over-downloading 640px images when a smaller generated image is enough.
- Decorative crescent glints are SVG paths, not SVG text, so Lighthouse should not treat them as LCP text.
- Decorative hero stars are CSS-drawn sparkle shapes with empty DOM nodes, not literal "*" text, so the visual star effect stays while LCP and agent parsers do not see the sky as content text.
- Mobile hero performance follow-up, 2026-06-20:
  - Desktop keeps the full animated hero star/crescent and subtle hero image zoom treatment.
  - At phone/tablet widths, app/globals.css disables the costly hero slide zoom, animated star filters, halo pulses, and white-flash loops.
  - Mobile keeps a premium static glow by scaling star sizes down, reducing visible star count, adding lightweight box-shadow glow, and containing hero sky paint work.
  - Home hero mobile type and CTA sizing were tightened so English and Arabic text stays polished inside the first viewport.
  - Validation used npm run build plus Playwright mobile checks for /?lang=en and /?lang=ar, with a desktop computed-style check confirming desktop animations remain active.
- Generic hotel grid cards no longer force the first two hotel images to high priority; the hero image keeps first-load priority, while below-fold hotel cards can load normally.
- The cart drawer is split out of the homepage/header initial bundle and loads only when the cart opens, keeping Ant Design date-picker/drawer work off the first mobile render.
- Google Analytics and Facebook Pixel keep early queue stubs, then load their external libraries after first interaction, tab hide, or a short delay. This preserves queued conversion calls while reducing initial mobile main-thread pressure.
- Hotel listing card images use a lower optimized quality setting for below-fold cards to reduce the remaining mobile image-delivery warning without changing layout.
- The support chat widget is loaded after idle unless chat query params are present, while keeping query-driven chat flows immediate.
- Analytics keeps lightweight GA/Facebook stubs early but defers third-party library downloads until after initial mobile rendering pressure has passed.
